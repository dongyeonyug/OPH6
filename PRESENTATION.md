## 1. 프로젝트 소개

TripCanvas는 나라를 먼저 고르고 그 나라의 여행 사진첩을 여는 로컬 우선 사진 앨범입니다. 첫 화면은 마케팅 페이지가 아니라 실제 세계 지도입니다. 사용자는 나라를 누르고, 오른쪽 패널 또는 모바일 하단 시트에서 사진 요약과 저장된 사진을 먼저 봅니다.

## 2. 작업지시 15점

보여줄 증거:

- `GOAL.md`: 제품 목표, 완료 조건, 점수표, 검증 루프가 명확합니다.
- `SPEC.md`: 기능 요구사항과 금지 범위를 분리했습니다.
- `CONTROL.md`: 배포 필수, 로컬 사진만 허용, 한국어 우선, 지도 우선 레이아웃 같은 운영 노브가 있습니다.

발표 문장:

"처음부터 기능 목록만 준 것이 아니라, 완료 조건과 검증 루프까지 에이전트가 추적하도록 작업지시를 구조화했습니다."

## 3. 자율 수행도 15점

보여줄 증거:

- `PLAN.md`: 단계별 실행 계획과 현재 상태.
- `ATTEMPTS.md`: React 19 충돌, Next 설정 오류, Playwright 실패, 모바일 hover preview 문제를 수정한 기록.
- GitHub 커밋: 구현과 검증 변경이 히스토리로 남습니다.

발표 문장:

"실패를 숨기지 않고 `ATTEMPTS.md`에 남긴 뒤 다음 조정으로 이어갔습니다. 이 문서가 자율 루프의 블랙박스 역할을 합니다."

## 4. 검증, 루프설계 10점

보여줄 증거:

- `npm run build`: 통과.
- `npm run qa:playwright`: 데스크톱과 모바일 Chromium에서 통과.
- 스크린샷: `test-results/desktop-tripcanvas.png`, `test-results/mobile-tripcanvas.png`.
- `QA.md`: 콘솔 오류, 오버플로, 지속성, 슬라이드쇼, 앨범, 사진 편집 결과 기록.

발표 문장:

"눈으로만 확인하지 않고, 브라우저 자동화가 실제 데모 동선을 반복하도록 만들었습니다."

## 5. 실시간 시연 완성도 25점

시연 순서:

1. 배포 URL을 엽니다.
2. 세계 지도에서 브라질 또는 일본을 선택합니다.
3. 오른쪽 패널 또는 모바일 하단 시트가 열리는 것을 보여줍니다.
4. `사진 추가`로 로컬 사진 파일을 저장합니다.
5. `사진첩 열기`에서 제목, 설명, 날짜, 태그, 카테고리를 수정합니다.
6. 대표 사진을 지정하고 큰 사진 보기를 엽니다.
7. `슬라이드쇼`를 열고 `1 / 1` 같은 인덱스를 확인합니다.
8. 새로고침 후 사진과 편집값이 유지되는 것을 보여줍니다.

## 6. 창의성 및 독창성 20점

TripCanvas의 차별점은 목록이나 대시보드가 아니라 지도에서 기억을 시작한다는 점입니다. 사진첩은 나라별로 분리되고, 데스크톱은 오른쪽 패널, 모바일은 하단 시트로 맥락이 유지됩니다. 또한 앱 안에는 대회 증거를 숨기고, `README.md`, `QA.md`, `PRESENTATION.md`에만 하네스 스토리를 둔 점이 제품 순도를 지킵니다.

## 7. 활용가능성 10점

확장 방향:

- 계정과 로그인 추가
- 실제 클라우드 업로드와 백업
- 공유 여행 앨범
- DB 기반 여행 저널
- 지도별 통계와 여행 루트
- 모바일 앱 또는 PWA 오프라인 모드

## 8. 발표 마무리

"TripCanvas는 작은 MVP지만, 실제 사용자가 데모에서 눌러볼 수 있는 지도, 로컬 사진 저장, 앨범 편집, 슬라이드쇼가 있습니다. 동시에 Codex가 계획, 시도, 검증, 배포를 어떻게 반복했는지 문서와 히스토리로 확인할 수 있습니다."
